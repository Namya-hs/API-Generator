Initially, log in to Azure DevOps using your developer (ex:abc@incadeadev.com) account. Then, navigate to the repository section by clicking on the corresponding option in the navigation bar. From there, locate and select the project labeled 'WAP-API-Component'. Finally, clone the project by clicking on the 'Clone' button. "Additionally, ensure that you select the 'develop' branch before proceeding with the cloning process."

  • Now, in 2nd step navigate to the directory where you cloned the project and open two specific folders: 'incadea.API Middleware' and 'incadea.API Adapter'.

  • In the 'incadea.API Middleware' folder, find the file named 'incadea.api.middleware.sln' and open it using Visual Studio or any other preferred integrated development environment (IDE), preferably Visual Studio 2022 Community version. Similarly, in the 'incadea.API Adapter' folder, locate and open the 'incadea.api.sln' file.

  • After opening either the Middleware or API Adapter solution file (.sln) in Visual Studio, navigate to the bottom of the Visual Studio interface. There, you'll notice the option to develop represented by a Git icon ( ). Click on this icon to access the branch selection menu. Choose the 'develop' branch from the list provided.

  • When creating a new branch, ensure it's based on the appropriate branch, typically 'develop', although this may vary. Follow the standard naming convention for feature branches, which should be in kebab case and lowercase. For example, use 'features/yourTaskId-name-of-your-task' format, where 'features' remains the prefix and 'yourTaskId-name-of-your-task' describes the task. Remember, consistency and adherence to these conventions are crucial.
